## Phase 0: Research

*See [research.md](./research.md) for full decisions.*

Key decisions resolved:

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | README uses `<!--start-->` / `<!--end-->` include markers | Matches existing `index.md` include pattern; hides badges/template-credit from docs site |
| D2 | README contains full quick-start prose; quickstart.md in docs expands it | DRY for the minimal path; docs page adds depth |
| D3 | `usage.md` documents all 7 `migrate_content()` params via prose table | Supplements auto-generated `api.md` docstring rendering |
| D4 | `configuration.md` replaces `dev/test/prod` with `source/destination` | Matches actual `config.yml`; eliminates confusion |
| D5 | `mkdocs.yml` nav order: Home → Quickstart → Usage → Configuration → API | Reading-order matches new-user journey |
| D6 | Two auth paths shown everywhere: profile (ArcGIS named store) and URL+username+password | Both are supported by `_connect_gis()`; profile takes precedence |
| D7 | No new MkDocs plugins required | All needed plugins already in `docsrc/requirements.txt` |
